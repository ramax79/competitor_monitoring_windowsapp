# Быстрый старт Desktop приложения

## Шаг 1: Установка зависимостей

```bash
cd desktop_app
pip install -r requirements.txt
```

## Шаг 2: Запуск backend

**Важно:** Backend должен быть запущен перед использованием приложения!

В корне проекта (не в `desktop_app/`):

```bash
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

## Шаг 3: Проверка подключения (опционально)

```bash
python test_connection.py
```

## Шаг 4: Запуск приложения

```bash
python main.py
```

## Сборка в .exe

### Windows (автоматически)

```bash
build.bat
```

### Вручную

```bash
pyinstaller CompetitorMonitor.spec --clean --noconfirm
```

Результат: `dist/CompetitorMonitor.exe`

## Использование

1. **Анализ текста:**
   - Вкладка "Текст конкурента"
   - Вставьте текст
   - Нажмите "Проанализировать"

2. **Анализ изображения:**
   - Вкладка "Изображение"
   - Нажмите "Выбрать изображение"
   - Выберите файл (макс. 5 МБ)
   - Нажмите "Проанализировать"

3. **Парсинг URL:**
   - Вкладка "URL конкурента"
   - Введите URL
   - Нажмите "Проанализировать"

4. **Просмотр истории:**
   - Кликните на любой элемент в списке "Последние запросы"
   - Результаты отобразятся в правой панели

## Настройка API адреса

Если backend запущен на другом адресе:

1. Установите переменную окружения:
   ```bash
   set API_BASE=http://your-server:8000
   ```

2. Или измените в `main.py`:
   ```python
   API_BASE = "http://your-server:8000"
   ```

## Устранение проблем

### "Backend недоступен"

- Убедитесь, что backend запущен
- Проверьте адрес в `API_BASE`
- Проверьте файрвол

### Ошибка при загрузке изображения

- Максимальный размер: 5 МБ
- Поддерживаемые форматы: PNG, JPG, JPEG, GIF, BMP

### Приложение не запускается

- Проверьте, что все зависимости установлены
- Запустите через консоль для просмотра ошибок
